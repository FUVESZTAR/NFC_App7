NFC App7 - Developer Guide

Overview
- Android app: /PlantNFC (Kotlin, Jetpack Compose, Hilt, Room, DataStore).
- Legacy/reference web implementation: /Nfcall.html.
- Main flow: load plants from Google Sheets -> generate NFC payload -> write/read tag -> save local records -> sync upload to Google Apps Script.

Architecture
- presentation/: Compose screens + ViewModels.
  - generator/: plant selection, NFC payload preview, NFC write, GPS packet.
  - reader/: NFC read and decode.
  - nfclist/: local NFC record list + sync actions.
  - settings/: configuration for Google Sheet and writer endpoint.
- domain/: models and repository interfaces.
- data/
  - remote/GoogleSheetsDataSource.kt: Google Sheets and Apps Script HTTP calls.
  - local/: Room entities, DAO, database.
  - repository/Repositories.kt: repository implementations and mapping.
- util/: preferences, encoding/decoding helpers.

Key features
1) Plant source loading
- Query Google Visualization endpoint (gviz/tq) from configured sheet.
- Loads active NFC plants (A-E columns where CR='Y').
- Stores plants in Room, then UI observes active list.

2) NFC generation
- User selects plant + variety, sets metadata (NFC ID/date/type).
- Payload generated by NfcTextCodec.
- Optional compressed GPS packet included.
- Link preview generated from selected plant ID.

3) NFC operations
- NfcBridge handles NFC write and read interactions.
- Reader screen parses payload fields for display/copy/open-link actions.

4) Local persistence
- Generated records saved to Room with sync status (PENDING/SYNCED/CONFLICT).
- List screen supports search and delete.

5) Cloud sync
- Upload: pending records POSTed to Apps Script endpoint.
- Last NFC ID fetched from writer endpoint GET response.
- Download/import behavior depends on backend support and repository implementation.

Configuration
- Stored via DataStore in AppPreferences:
  - plant_sheet_id
  - plant_sheet_name
  - nfc_writer_url
  - nfc_writer_secret
  - language

Important behavior notes
- Plant loader expects gviz JSON wrapped in Query.setResponse(...).
- Parser maps plant fields robustly using headers and fallback column positions.
- Startup refresh and manual refresh report failures via snackbar messages.
- NFC list sync actions should handle Result success/failure explicitly in ViewModel.

Build
- From /PlantNFC:
  - chmod +x gradlew
  - echo "sdk.dir=$ANDROID_HOME" > local.properties
  - ./gradlew lintDebug testDebugUnitTest assembleDebug
